타입 분류 로직 — 모든 응답자 매핑 (우선순위)

우선순위 순서로 첫 매칭 적용. 마지막 규칙이 fallback이므로 모든 참가자가 반드시 A·B·C 중 하나로 분류됨.

A

의사결정 단계 리더

우선순위 1
Q4 = "현재 검토 중"
OR Q4 = "이미 도입해서 이용 중"
OR Q3 = "Ent 구축형"
OR Q5 ∈ {보안·망분리, 그룹웨어·ERP 연동}

현장 브로셔: AI 도입 ROI·의사결정 가이드
B

실무 효율 혁신가

우선순위 2
(A에 해당하지 않는 경우)
Q2 = "플로우+AI" 또는 "플로우+위키"
OR Q5 ∈ {프로젝트·업무 관리, 보고·결재, 데이터 중앙화, 재택·하이브리드, 외부 협업}

현장 브로셔: 워크플로우·AI 활용 사례집
C

탐색·인지 단계

우선순위 3 (fallback)
위 어디에도 매칭되지 않는 모든 응답자
예: Q4="아직 계획 없음" + Q2="플로우" + Q5="기타/메신저 통합/비용 절감"

현장 브로셔: 플로우 소개·AX 트렌드 리포트
분류 함수 의사코드
function classify({q1, q2, q3, q4, q5}) {  // q5 = Array (복수 선택)
  // 우선순위 1: 의사결정에 가까운 신호
  if (q4 === '현재 검토 중' || q4 === '이미 도입해서 이용 중') return 'A';
  if (q3 === 'Ent 구축형') return 'A';
  if (q5.includes('IT 보안·망분리 환경 협업') ||
      q5.includes('그룹웨어·ERP 연동')) return 'A';

  // 우선순위 2: 실무 활용 의도
  if (q2 === '플로우+AI' || q2 === '플로우+위키') return 'B';
  if (q5.some(v => ['프로젝트·업무 관리 고도화','보고·결재 프로세스 디지털화',
                    '업무 데이터·지식 중앙화','재택·하이브리드 근무 환경',
                    '외부 협업 (고객사·파트너사)'].includes(v))) return 'B';

  // 우선순위 3: 그 외 모두 (탐색 단계)
  return 'C';
}
※ 이 로직은 config/classifier.ts 한 파일에 분리. 행사 당일에도 즉시 교체 가능.
